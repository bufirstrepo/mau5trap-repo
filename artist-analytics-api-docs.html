Overview

The Artist Analytics API provides real-time access to comprehensive artist performance data including streaming statistics, social media metrics, tour analytics, and revenue insights. Built for record labels, artist managers, and music industry professionals.

Base URL

https://api.artistanalytics.io/v2

Key Features

  • Real-time streaming data from Spotify, Apple Music, and YouTube
  • Social media growth tracking across Instagram, TikTok, and Twitter
  • Tour performance metrics and ticket sales analytics
  • Revenue projections and royalty tracking
  • Custom date ranges and data aggregation
  • Webhook notifications for milestone achievements

Authentication

All API requests require authentication using API keys. Include your API key in the request header:

Authorization: Bearer YOUR_API_KEY
Getting Started: Request an API key from your account dashboard at dashboard.artistanalytics.io

Example Request

curl -X GET "https://api.artistanalytics.io/v2/artists" \
  -H "Authorization: Bearer sk_live_abc123xyz456" \
  -H "Content-Type: application/json"

Endpoints

Artists

GET /artists

Retrieve a list of all artists under your management.

Query Parameters

Parameter Type Description
limit integer Number of results (default: 20, max: 100)
offset integer Pagination offset (default: 0)
sort string Sort by: name, streams, followers (default: name)

Response

  "data": [

      "id": "art_abc123",
      "name": "Luna Sterling",
      "genre": "Electronic Pop",
      "total_streams": 45893021,

      "followers": {
        "spotify": 892341,
        "instagram": 1453920,
        "tiktok": 3294012
      },
      "verified": true
    }
  ],
  "pagination": {
    "limit": 20,
    "offset": 0,
    "total": 47

}

Webhooks

POST /webhooks

Create a webhook to receive real-time notifications.

Request Body

{
  "url": "https://yourapp.com/webhook",
  "events": [
    "milestone.streams",
    "milestone.followers",
    "tour.sellout"
  ],
  "artist_ids": ["art_abc123"],
  "active": true
}

Available Events

  • milestone.streams - Stream count milestones (1M, 10M, etc.)
  • milestone.followers - Follower milestones
  • tour.sellout - Show sells out
  • viral.detected - Content goes viral
  • chart.entry - Enters charts

Error Handling

The API uses standard HTTP status codes and returns detailed error messages in JSON format.

Error Response Format

{
  "error": {
    "code": "invalid_artist_id",
    "message": "The provided artist ID does not exist",
    "status": 404,
    "timestamp": "2024-10-15T14:32:18Z"
  }
}

Common Status Codes

Code Description
200 Success - Request completed successfully
400 Bad Request - Invalid parameters or malformed request
401 Unauthorized - Invalid or missing API key
403 Forbidden - Valid credentials but insufficient permissions
404 Not Found - Resource doesn't exist
429 Too Many Requests - Rate limit exceeded
500 Internal Server Error - Something went wrong on our end

Rate Limits

To ensure fair usage and system stability, the API implements rate limiting.

Rate Limits by Plan:
  • Free: 100 requests/hour
  • Professional: 1,000 requests/hour
  • Enterprise: 10,000 requests/hour

Rate Limit Headers

Every API response includes rate limit information in the headers:

X-RateLimit-Limit: 1000
X-RateLimit-Remaining: 847
X-RateLimit-Reset: 1697389200
Note: When you exceed the rate limit, the API returns a 429 status code. The Retry-After header indicates when you can retry the request.
